# Style Docs

This directory documents coding-style guidance for the repository.

It currently includes both repo-wide style guidance and frontend CSS system guidance.

Start here:

- [Repository Coding Style](./repository-coding-style.md)
- [Backend Python Modularization](./backend-python-modularization.md)
- [How To Document Style Changes](./how-to-document-style-changes.md)

Frontend CSS-specific docs:

- [Unified CSS System](./unified-css-system.md)
- [How To Extend Unified CSS](./how-to-extend-unified-css.md)

The goal of this directory is to keep future agents aligned on style decisions that should outlive a single change.

When a user sets a new coding-style boundary, this directory is where that rule should usually be documented.
